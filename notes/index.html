deploy.sh

This is my deploy.sh file:

	#!/bin/bash -e
	J="$2"
	if [ -f "$J/site.min.js" ];
	then
	  rm $J/site.min.js
	fi
	allFiles=($J/*.js)
	length=${#allFiles[@]}
	for ((i=0; i<$length; i++))
	do
	   cat ${allFiles[$i]} >> $J/site.js
	done
	java -jar /usr/share/java/yuicompressor-2.4.8.jar -o $J/site.min.js $J/site.js
	if [ -f "$J/site.js" ];
	then
	  rm $J/site.js
	fi
	echo "js minified"

	C="$3"
	if [ -f "$C/site.min.css" ];
	then
	  rm $C/site.min.css
	fi
	allFiles=($C/*.css)
	length=${#allFiles[@]}
	for ((i=0; i<$length; i++))
	do
	   cat ${allFiles[$i]} >> $C/site.css
	done
	java -jar /usr/share/java/yuicompressor-2.4.8.jar -o $C/site.min.css $C/site.css
	if [ -f "$C/site.css" ];
	then
	  rm $C/site.css
	fi
	echo "css minified"

	commit_message="$1"
	git add .
	git commit -a -m "$commit_message"
	git push origin master

This file can be run from git bash using a command like this: deploy.sh [commit message] [js folder location] [css folder location] so like this deploy.sh "Update homepage" assets/js assets/css. It pushes to a GitHub repository which has a webhook to ping my deploy.php on a Git push. Here is my deploy.php page:

	include 'deploy.conf';
	if(is_dir($fol)){
		shell_exec("rm -rf ".$fol);
	}
	copy("https://github.com/".$username."/".$repos."/archive/".$branch.".zip", "master.zip");
	shell_exec("unzip master.zip");
	unlink("master.zip");
	shell_exec("mv ".$repos."-master ".$fol);

and the deploy.conf looks something like this:

	$username = "wlangford";	//Your GitHub username
	$repos = "personal";		//Your repository name
	$branch = "master";			//Your branch you'd like us to use
	$fol = "site";				//The folder the site will be saved in
